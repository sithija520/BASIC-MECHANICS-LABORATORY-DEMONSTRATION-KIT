Software & Hardware Setup Instructions

Follow these instructions to set up the connection between your Arduino and the web interface, and to interface with various sensors and actuators.

Setting Up the Real-Time Data Connection

Follow these steps to install the necessary software and create the 'bridge' between your Arduino sensors and the web interface.

First, you need to install the software that makes everything work.

  • Install Python: This is the programming language we'll use for the "bridge".
  • Install Arduino Software (IDE): This is what you'll use to program your Arduino board.
  • Install Python Helpers: Open your computer's terminal or command prompt and run this command. It gives Python the tools it needs to talk to the Arduino and the website. 
pip install pyserial websockets

Next, you need to tell your Arduino what to do.

  • Write the Arduino Code: Create the code that reads the information from your potentiometer and encoder.
  • Upload the Code: Connect the Arduino to your computer with a USB cable and use the Arduino software to send this code to the board. Now, the Arduino will constantly send sensor data through the USB cable.

Now, you need a program that acts as a link between your Arduino and your website. This program's job is to:

  • Listen for the data coming from the Arduino through the USB port.
  • Create a "local server" that your website can connect to.
  • Pass the Arduino data along to the website as soon as it gets it.

You're ready to see it all work together.

  • Run the Python Link: Start your Python program. It will begin listening for the Arduino.
  • Connect the Website: Open your project's webpage in a browser and click the "Start" or "Connect" button.

That's it! The website is now connected to the Python link, which is connected to your Arduino. You should see the sensor information appear on your screen.

Technical Guide: Interfacing a Load Cell with Arduino

  • Arduino Uno
  • 5kg Bar-Type Load Cell
  • HX711 Amplifier Module
  • Pulley or Roller Bearing

A. Load Cell to HX711 Module:

  • Red Wire → E+
  • Black Wire → E-
  • White Wire → A-
  • Green Wire → A+

B. HX711 Module to Arduino Uno:

  • GND → Arduino GND
  • VCC → Arduino 5V
  • DT → Arduino Digital Pin 2
  • SCK → Arduino Digital Pin 3

  • Install Library: In the Arduino IDE, go to the Library Manager and install "HX711_ADC by Olav Kallhovd".
  • Calibrate: Upload a calibration sketch to read raw data from the sensor.
    • With no weight, allow the sensor to tare (zero out).
    • Place a known mass (e.g., 500g) on the sensor and record the raw reading from the Serial Monitor.
    • Calculate the Calibration Factor using the formula:
      Calibration Factor = Raw Reading / Known Mass (in grams)
  • Measure: Use a measurement sketch.
    • Set the calculated Calibration Factor in the code.
    • The Arduino will now output the measured force in grams.

The load cell reading provides a proxy for belt tension.

  • Mounting: Securely mount the load cell onto a fixed frame.
  • Force Application: Attach the pulley to the active end of the load cell.
  • Measurement: Position the pulley to press against the belt, causing a slight deflection. The force measured by the load cell is directly proportional to the tension in the belt. A higher force reading indicates higher tension.

Technical Guide: CAM Analysis System

  • Arduino Uno
  • Slide Potentiometer (Fader)
  • Rotary Encoder (Incremental)
  • CAM System with Follower
  • Connecting Wires and Mechanical Couplers/Linkages

Proper mechanical integration of the sensors is critical for accurate measurements.

Slide Potentiometer (For Displacement):

  • Mount the body of the potentiometer to the fixed frame of the CAM system.
  • Mechanically connect the slider of the potentiometer directly to the CAM follower. A small, rigid linkage may be used.
  • Requirement: The slider must travel the exact same distance as the follower moves.

Rotary Encoder (For Angle):

  • Couple the shaft of the encoder to the main rotating shaft of the CAM, ensuring a 1:1 rotation ratio.
  • Mount the body of the encoder so that it remains stationary.
  • Requirement: When the CAM shaft rotates 360°, the encoder shaft must also rotate exactly 360°.

Connect the sensors to the Arduino Uno as follows.

Slide Potentiometer → Arduino:

  • VCC/5V pin → Arduino 5V
  • GND pin → Arduino GND
  • OUT/Signal pin → Arduino Analog Pin A0

Rotary Encoder → Arduino:

  • VCC/5V pin → Arduino 5V
  • GND pin → Arduino GND
  • CLK (Pin A) → Arduino Digital Pin D2 (must be an interrupt pin)
  • DT (Pin B) → Arduino Digital Pin D3

These are the principles for the Arduino code logic.

Follower Displacement (from Slide Potentiometer):

  • Use the analogRead(A0) function to get a raw value between 0-1023.
  • Map this raw ADC value to the actual physical displacement in millimeters (mm).
  • Calibration: First, measure the follower's maximum travel distance (max_displacement_mm). Then, record the minimum and maximum raw ADC values from the potentiometer (min_ADC, max_ADC) that correspond to this travel.
  • Formula: Displacement (mm) = map(analogValue, min_ADC, max_ADC, 0, max_displacement_mm);

CAM Angle (from Rotary Encoder):

  • Using interrupts is essential for accuracy. Use attachInterrupt() on pin D2.
  • The interrupt will trigger on every change of the CLK pin (e.g., on the RISING edge) as the CAM rotates.
  • Inside the interrupt routine, read the state of the DT pin (D3) to determine the direction of rotation (clockwise or counter-clockwise).
  • Increment or decrement an angle counter variable based on the direction.
  • Convert the counter value to degrees (°) using the encoder's PPR (Pulses Per Revolution) specification.
  • Formula: Angle (°) = (counter / PPR) * 360;

Finally, the Displacement and Angle data pairs acquired at each step can be sent via WebSockets to be rendered on your website's graph.

A Complete Guide to Using a Stepper Motor with Arduino and an L298N Driver

This guide provides a full, step-by-step process for connecting and running a NEMA 17 stepper motor at a constant speed using an Arduino Uno and an L298N driver module.

First, make sure you have all the necessary parts:

  • Arduino Uno board
  • NEMA 17 Stepper Motor (typically with 4 wires)
  • L298N Motor Driver Module
  • External Power Supply (9V or 12V) for the motor
  • Jumper Wires
  • USB Cable

This is a critical first step. The four wires from the motor belong to two separate internal coils (Coil A and Coil B). You must identify which wires belong together.

The Easiest Method: With the motor not connected to any power, take any two wires and touch their metal tips together. Now, try to turn the motor's shaft with your hand.

  • If the shaft becomes harder to turn, you have found a pair belonging to the same coil.
  • If it turns easily, they are not a pair. Try a different combination.

Repeat this process until you have identified the two pairs of wires.

Connect all the components correctly as described below.

Motor to L298N Driver:

  • Connect the first wire pair (Coil A) to the OUT1 and OUT2 terminals on the L298N.
  • Connect the second wire pair (Coil B) to the OUT3 and OUT4 terminals.

Arduino to L298N Driver:

  • Arduino Digital Pin 8 → L298N IN1
  • Arduino Digital Pin 9 → L298N IN2
  • Arduino Digital Pin 10 → L298N IN3
  • Arduino Digital Pin 11 → L298N IN4

Power Supply and Common Ground (Very Important):

  • Connect the positive (+) wire of your external power supply to the 12V terminal on the L298N.
  • Connect the negative (-) wire of your external power supply to the GND terminal on the L298N.
  • Run another jumper wire from the GND terminal on the L298N to any GND pin on your Arduino. This common ground connection is essential for the system to work.

An Arduino program tells the motor what to do. You don't need to write complex code because a built-in library handles the hard work.

  • The program uses the Stepper.h library, which is included with the Arduino software.
  • First, you tell the program which Arduino pins (8, 9, 10, 11) are connected to the driver's input pins.
  • Next, you use a single command, setSpeed(), to define how fast the motor should turn. You give it a number that represents the speed in RPM (Revolutions Per Minute). For example, setting a speed of 60 will make the motor complete one full turn every second.
  • Finally, the program continuously tells the motor to move one step forward in a loop. Because the loop runs very fast, this results in smooth, constant rotation.

  • Connect your Arduino board to your computer using the USB cable.
  • In the Arduino IDE software, go to Tools > Board and select "Arduino Uno."
  • Go to Tools > Port and select the COM port that your Arduino is connected to.
  • Click the Upload button (the arrow icon pointing to the right).

  • After the program has successfully uploaded, keep the USB cable connected.
  • Now, turn on the external power supply that is connected to the L298N driver.
  • The stepper motor should immediately begin to rotate at the constant speed you defined in the program.

Displaying Load Cell Data on an I2C LCD: A Technical Guide

This guide outlines the process for connecting a load cell and an I2C LCD display to an Arduino Uno to show live weight readings.

  • Arduino Uno
  • Load Cell with HX711 Amplifier Module
  • 16x2 LCD Display with an I2C Backpack Module
  • Jumper Wires

Connect all components to the Arduino. The I2C module on the LCD simplifies its connection to just four wires.

Load Cell Amplifier (HX711) to Arduino:

  • VCC → Arduino 5V
  • GND → Arduino GND
  • DT → Arduino Digital Pin D2
  • SCK → Arduino Digital Pin D3

I2C LCD Display to Arduino:

  • VCC → Arduino 5V
  • GND → Arduino GND
  • SDA → Arduino Pin A4
  • SCL → Arduino Pin A5

Note: Both the HX711 module and the LCD share the Arduino's 5V and GND pins. A small breadboard can be used to easily distribute these connections. A common ground (GND) between all components is essential.

The Arduino program coordinates the two main tasks: reading the sensor and displaying the data. It does this by following these logical steps:

  • Use Special Libraries: The program relies on two key libraries: one to communicate with the HX711 load cell amplifier and another to control the I2C LCD display.
  • Find the LCD Address: Before running the main program, a special "I2C Scanner" sketch is used to automatically detect the unique address of your specific LCD module. This address is then used in the main program to establish communication.
  • Use the Calibration Factor: The program uses the calibration_factor that you determined in the earlier steps. This crucial number is used to convert the raw, abstract signal from the load cell into a meaningful weight, such as grams (g).
  • The Main Loop: Once set up, the program runs a continuous loop that does the following:
    • Reads the current weight from the load cell.
    • Clears any old text from the LCD screen.
    • Prints a static label, like "Weight:", onto the first line of the display.
    • Prints the live weight reading (the number in grams) onto the second line of the display.
    • Waits for a brief moment (e.g., half a second) and then repeats the entire process, providing a constantly updated display.
